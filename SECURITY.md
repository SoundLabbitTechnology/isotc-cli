# Security Policy

## サポート対象バージョン

| バージョン | サポート状況 |
|-----------|-------------|
| 0.2.x     | サポート中   |
| 0.1.x     | サポート終了 |

## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、**GitHub Issues を公開せず**、以下の方法で報告してください。

### 報告方法

- **メール**: 組織のセキュリティ窓口（公開連絡先が設定されている場合）
- **GitHub Security Advisories**: [Security] タブから「Report a vulnerability」を選択

### 報告に含める情報

- 脆弱性の種類と影響範囲
- 再現手順
- 想定される影響
- 修正案（あれば）

### 対応方針

- 報告を受けてから 48 時間以内に受領確認の返答を行います
- 重大な脆弱性については、修正の完了とリリース後に公開報告を行います
- 報告者には適切なクレジット表記を行います（希望される場合）

## セキュリティ上の注意

- `.spec/constitution.toml` 等の設定ファイルに API キー等の機密情報を保存しないでください
- 環境変数経由での機密情報の注入を推奨します
